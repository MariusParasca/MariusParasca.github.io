1. Descrierea proiectului

Ideea aplicatiei este de a concepe un magazin web de accesorii de imbracaminte pentru acoperit crestetul capului. Utilizatorul are posibilitatea de a proba accesoriile fara a necesita prezenta fizica folosind realitatea imbogatita (AR – Augmented Reality). Pe langa lista de accesorii valabile in magazinul online, utilizatorului ii vor fi oferite o lista de recomandari:

  • In functie de sezon (bazata pe data)
  • In functie de moda curenta (locatie)
  • In functie de preferintele specifice utilizatorului (wishlist)
  • Eventual pe baza cercului de prieteni intr-o retea sociala

2. Idei/concepte importante privind designul general

Avand o idee generala in gand am incercat sa transpunem design-ul aplicaitei pe primele noastre schite, precum se vede in partea stanga a pagini. In partea dreapta putem observa implementarea acestor schite, cu mici modificari, in HTML si CSS, care reprezinta design-ul final.

3. Maniera de interacţiune cu utilizatorul

Utilizatorul acceseaza initial pagina principala a aplicatiei unde poate vedea ultimele accessorii adaugate, si poate accessa toate categoriile accessoriilor prin dropdown-ul din partea stanga sus. Fiecare accesoriu isi are pagina lui, unde utilizatorul poate proba "virtual" accesoruiul dat folosind realitatea augmentata. Proband accesoriul, utilizatorul poate salva o poza cu accesoriul dorit. Utilizatorii mai au posibilitatea de a adauga accesoriile dorite intr-un cos, simuland astfel magazinul online. De asemenea fiecare utilizator are o pagina separata pentru accesoriile sale preferate.

5. Arhitectura software de ansamblu

Single Page Application

Site-ul este o aplicatie WEB de tipul SPA - Single Page Application. Beneficiul acestei categorii de aplicatie este interactiunea utilizatorului cu aplicatia, oferind-ui o experienta similara cu aplicatiile de tip desktop. Accesand diferite pagini fereastra utilizatorul nu se reincarca, iar pagina dorita se schimba aproape instant. Un dezavantaj al acestei categorii este request-ul initial. Acesta contine toate fisierele site-ului, fiind mai mare in dimensiune si poate dura mai mult timp decat un request simplu.

Structura consta intr-un fisier inital index.html care contine template-ul de baza a aplicatiei, si in care sunt incarcate dinamic celelalte pagini html. Rutarea paginilor se face in javascript la nivel de client pe baza variabilei window.location.hash (ceea ce vine dupa # in url).

tracking.js

Pentru partea de AR - Augmented Reality folosim libraria tracking.js (https://trackingjs.com/), care aduce diferiti algoritmi si tehnici din domeniul computer vision direct in mediul browserului. Cu ajutorul acesteia vom detecta fata utilizatorului. Pentru a accesa camera web a utilizatorului, folosim tag-ul html video. Specificam librariei ca folosim camera web si o setam pentru detectia faciala. Aceasta la randul ei returneaza coordonatele fetei gasite, pe care le folosim sa punem accesoriul (pentru acoperit crestetul capului) pe un canvas ce se afla peste tag-ul video, astfel simuland "probarea" accesoriului. De asemenea odata ce utilizatorul probeaza accesoriul are posibilitatea de a face o captura de ecran (screenshot) a sa (din camera web) si accesoriul probat. Pentru a obtine asta folosim mediaDevices.getUserMedia ce face parte din API-ul navigator.

Local storage

Avand posibilitatea de a-si face o captura de ecran cu accesoriul dorit, utilizatorul poate salva aceste imagini local, fiind accesibile pentru descarcare dupa capturare. Pentru a obtine aceasta functionalitate, folosim mecanismul de stocare locala a datelor oferit de Web Storage API: localStorage. Acesta mentinand datele local chiar si cand utilizatorul inchide browserul. La fel in local storage vor fi salvate accesoriile favorite ale utilizatorului.

Session storage

Un alt mecanism de stocare a datelor oferit de Web Storage API este sessionStorage. Ce il face diferit fata de precedentul e faptul ca valabilitatea datelor dureaza exact cat sesiunea paginii, in alte cuvinte, cat timp utilizatorul tine fereastra pornita. Functionalitatea acestuia o folosim pentru a simula cosul de cumparaturi, astfel simuland si magazinul online. Utilizatorul isi poate verifica cosul dupa selectia accesoriilor dorite, si la urmatoarea sesiune nu va avea in cos accesoriile din sesiunea trecuta, imbunatatind experienta utilizatorului.

Google Firebase

Neavand nevoie de un server, am cautat solutii de stocare a datelor astfel incat sa putem lua datele doar prin requesturi AJAX. Asadar am folosit Google Firebase ca baza de date NoSQL. Acolo am stocat toate datele legate de produsele pe care le ofera site-ul. Am stocat atat datele text cat si imaginile. Google Firebase ofera un mod de a interoga baza de date credand query-uri pe baza de functii putand chiar si filtra pe cloud datele de care avem nevoie.

Microdate

Libraria pe care o vom folosi pentru a marca datele site-ului nostru este Schema.org. Aceasta ajuta la intelegerea mai bine a continutului site-ului de catre motoarele de cautare(SEO - Search negine Optimization). Marcatorul parinte pe care il folosim este schema store reprezentand magazinul schema generala pentru magazine. De exemplu vom folosi prorietatea offer pentru fiecare produs pe care il ofera site-ul.
